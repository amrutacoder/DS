What is RMI?
RMI = Remote Method Invocation
It is a Java technology that allows:
one Java program to call methods of another Java program running on different machine/system.
________________________________________
Simple Meaning
Client can call server methods remotely.
Like:
Client → calls add() method → Server executes → Result returned
________________________________________
Real Life Example

•	You enter numbers 
•	Server processes 
•	Result comes back 
________________________________________

File	Purpose

↓

↓

RMI Registry is:
a naming service used to store and locate remote objects

Assignment 4: MPI (Message Passing Interface)
To find the sum of N elements in an array by distributing elements among n processors, and computing partial (intermediate) sums at each processor.
MPI means Message Passing Interface
MPI (Message Passing Interface) is a standard programming model used for communication between multiple processes in a distributed system.
It is used in distributed systems where:
•	Each processor works separately 
•	They communicate using messages 
•	Each processor computes part of the work
Idea of This Program
Assume:
•	Array = [1, 2, 3, 4, 5, 6] 
•	Processors = 3 
Then:
Processor	Data	Partial Sum
P0	1, 2	3
P1	3, 4	7
P2	5, 6	11
👉 Final Sum = 3 + 7 + 11 = 21
Input
•	Number of elements (N) 
•	Array elements 
•	Number of processors (n) 
________________________________________
📤 Output
•	Data assigned to each processor 
•	Intermediate sum at each processor 
•	Final sum

MPI.COMM_WORLD?
👉 MPI.COMM_WORLD is the default communicator in MPI.
It represents:
✔ All processes running your program
So if you run:
mpjrun.sh -np 4 mpisum

COMM_WORLD = {P0, P1, P2, P3}

Assignment 5: Clock Synchronization
Berkeley Algorithm for Clock Synchronization
What is Clock Synchronization?
In distributed systems:
👉 Every computer has its own clock
👉 But clocks are NOT perfectly same
👉 So time may differ in each node
Node A → 10:00  
Node B → 10:05  
Node C → 09:58
👉 All show different time ❌
👉 This creates confusion in system
________________________________________
🟢 Solution → Berkeley Algorithm
👉 It is used to synchronize all clocks to a common time
________________________________________
🧠 Main Idea
✔ One node becomes Master (Coordinator)
✔ It collects time from all nodes
✔ It calculates average time
✔ It sends correction to all nodes

Assignment: Mutual Exclusion (Token Ring Algorithm)
To implement the Token Ring Mutual Exclusion algorithm, where only one process at a time can enter the Critical Section using a circulating token.
What is Mutual Exclusion?
Mutual exclusion means:
Only one process can use a shared resource at a time (A shared resource is a resource that multiple processes want to access.)
Other processes must wait
Only one person can use ATM at a time
Token Ring Algorithm idea:
•	All processes are arranged in a ring (circle) 
•	A special message called TOKEN circulates in the ring 
•	Only the process holding the TOKEN can enter the critical section 
•	After finishing, it passes the TOKEN to the next process 
“Whoever has the token gets permission to use the resource.”

Real-life Analogy
Think of a microphone in a group discussion:
•	Only the person holding mic can speak (Critical Section) 
•	After speaking, mic is passed to next person (Token passing)

Assignment: Election Algorithms
Leader Election using Bully and Ring Algorithm
Leader Selection 
In a distributed system:
👉 There are many nodes
👉 One node must act as leader (coordinator)
👉 If leader fails → new leader must be selected
Why election is needed?
✔ Coordination of tasks
✔ Avoid conflict
✔ Manage system properly
Bully algorithm 
Bully Algorithm is a leader election algorithm in distributed systems
👉 Used when current coordinator (leader) fails
Each process has a unique ID
✔ Highest ID process becomes leader
✔ If a process detects failure, it “calls higher processes”
✔ Highest alive process “wins” the election (bully behavior)
P1, P2, P3, P4, P5
👉 P5 is initial leader
P5 (Leader) → FAIL ❌
Step 2: P2 detects failure
P2 starts election:
P2 → sends message to P3, P4, P5
Step 3: Responses
P3 → alive  
P4 → alive  
P5 → no response (dead)
Higher ID always wins → "Bully wins the fight"
Ring Algorithm
Ring Algorithm is a leader election algorithm in distributed systems
👉 All processes are arranged in a logical circle (ring)
👉 Election message moves around the ring
Highest ID process becomes leader
